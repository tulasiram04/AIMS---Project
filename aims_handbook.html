📋 Project Overview

Understanding the problem and AIMS solution space

💡 What is AIMS? AIMS (AI-Powered Multi-Agent Inventory Reconciliation System) is an enterprise-grade governance platform that automates the audit process by matching authoritative baseline CMDB records (CSV) against discoverable live network scanner snapshots (JSON) using LangGraph multi-agent workflows.

In modern enterprise IT environments, configuration drifts, rogue network switch deployments, and decommissioned ghost VMs introduce security exposure and audit compliance penalties. Manual reconciliation is error-prone and slow. AIMS solves this by performing real-time analysis, compiling formal ReportLab PDF audit summaries, and providing a context-aware chatbot for querying infrastructure compliance postures.

🤖 Multi-Agent AI Graph

Inside the agentic reconciliation pipeline (LangGraph + Gemini)

Instead of sending raw file sheets to a monolithic prompt context (which degrades accuracy and exceeds tokens), AIMS uses an organized LangGraph graph pattern to process discrepancies through specialized, state-mutating nodes:

🔍

1. Discovery Agent Node

Pre-processing and Matching

Parsed baseline CSV records + network scanner JSON.
Constructs ID-based maps of assets. Associates exact matches, and isolates unique nodes not present in one of the sources.
Pre-mapped match buckets sent to the analysis stage.
🧠

2. Drift Analysis Agent Node

Attribute Compliance Checking

Paired baseline/live records.
Instructs Gemini API to inspect configuration differences (e.g. CPU capacity, environment tags, software versioning).
Flags exact fields containing drift (e.g. Name drifted or config settings mismatched).
⚠️

3. Severity Classification Agent Node

Risk Exposure Engine

Identified discrepancies catalog.
Determines the severity ranking (CRITICAL, HIGH, MEDIUM, LOW) based on the asset role (e.g. production database vs staging container).
Assigned severity labels and risk scores per finding.
🛡️

4. Remediation & Recommendations Agent Node

Remediation Planner

Categorized and rated findings.
Formulates action plans, estimates the remediation effort (in hours), and calculates anticipated risk reduction.
Remediation guidelines and actionable next steps.
📝

5. Executive Summary Agent Node

Executive Report Structurer

All calculated findings and recommendations.
Synthesizes results into a high-level narrative summary. Prepares metrics (total assets, accuracy percentage, total issues).
Cohesive, readable executive brief sent directly to the database and PDF compilation trigger.

Tables and core fields modeling the inventory metadata

users

id, username, email, hashed_password, role, is_active, status

Tracks user access credentials and RBAC status roles (admin, analyst, auditor, viewer).

📂

id, filename, upload_type, total_records, upload_date

Logs uploaded inventory sources, capturing filenames and baseline records count.

id, asset_id, asset_name, asset_type, location, configuration (JSON)

Persistent table representing authoritative CMDB rows or scanner inventory items.

🔄

reconciliations

id, status, missing_assets_count, untracked_assets_count, recommendations (JSON)

Summary of each reconciliation scan. Holds overall score metrics and AI text summaries.

🔍

id, reconciliation_id, discrepancy_type, severity, details, business_impact

Individual findings detailing drifted values, severity classes, and recommended actions.

📄

id, reconciliation_id, file_path, executive_summary, generated_at

Tracks compiled ReportLab PDF artifacts and their physical server volume paths.

📜

audit_logs

id, user_id, action, action_category, details, username

Maintains high-integrity records of user actions, enabling complete system audit trails.

⚡ Installation & Setup

Launch and bootstrap instructions step-by-step

1

Setup Environment Config

Copy the environment template from the root folder into the backend directory and configure the variables:

# Run at project root directory
cp .env.example backend/.env

Open backend/.env in your editor and insert your Gemini API Key:

GEMINI_API_KEY=AIzaSyYourGeminiAPIKeyHere
JWT_SECRET_KEY=AIMS_Strong_JWT_Secret_For_Auth_Token_Generation
DATABASE_URL=postgresql://postgres:postgres@db:5432/inventory_db
2

Method A: Run with Docker (Recommended)

AIMS provides a unified docker compose stack that configures PostgreSQL, FastAPI backend, and Nginx frontend in a single step:

# Run at Project/ folder containing docker-compose.yml
docker compose up --build -d

Access the web application container at: http://localhost

3

Method B: Run Locally (Development Setup)

To run individual components outside Docker, start PostgreSQL or edit DB connection URLs in backend/.env to point to SQLite: 

# Terminal 1: Launch Backend
cd backend
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# Terminal 2: Launch Frontend
cd frontend
npm install
# Set target proxy to local FastAPI port
echo "VITE_API_URL=http://localhost:8000/api/v1" > .env.local
npm run dev

Access the development web application at: http://localhost:5173

🌐 Centralizing in the Cloud: If you are deploying to a shared public domain and want all user interactions stored in a central repository:
  1. Create a PostgreSQL database instance on a cloud provider like Supabase or Neon.tech.
  2. Edit your backend/.env configuration file and set DATABASE_URL to your new remote cloud database string (e.g., postgresql://user:pass@host.neon.tech/dbname?sslmode=require).
  3. Restart your docker compose services. The tables will automatically migrate on the cloud server.
  4. To manage the remote DB, open your local Adminer on port 8080, change the Server field to the remote host address (e.g., host.neon.tech), and enter your cloud credentials.

🔧 Operations & Troubleshooting

Quick resolutions for common setup issues

Observed Problem Root Cause Remediation Action
Backend fails database connection checks on startup Database initialization delay. The API server starts before PostgreSQL has completed creating the schema. Wait 30-40 seconds for the database schema creation to finish, then trigger backend service container restart.
AI features throw API Quota Errors The configured Gemini API Key is missing, inactive, or has exceeded its request limits. Add or rotate your GEMINI_API_KEY value inside the backend/.env file, and restart the backend.
PDF downloads as UUID or unnamed file CORS mapping error. The frontend cannot access the backend Content-Disposition header. Ensure expose_headers=["Content-Disposition"] is configured in CORSMiddleware, and use the centralized downloadService.ts for downloads.
JWT Login errors after database resets Stale user session tokens cached in the browser's local store. Clear localStorage from the DevTools Application tab, reload the tab, and login using default credentials.
